*자바의 메모리 모델과 Object 클래스

-자바 가상머신의 메모리 모델
: 운영체제의 관점에서는 자바 가상머신은 그냥 프로그램중 하나일 뿐.
: 따라서, 운영체제가 일반 프로그램에 4G의 메모리 공간을 할당한다면, JVM에게도 4G 메모리 공간을 할당해준다.

-자바 가상머신 메모리 영역 구분
: 효율성을 높이기 위해 세 개의 영역으로 구분한다.
>메소드 영역, Method Area
: 메소드가 올라가는 영역(메소드의 바이트코드)
: static 변수, static 변수는 쓰지 않아도 초기화까지 해버림.
: 저장하는 두 종류의 특징, 한 번 할당되면 프로그램이 종료될 때 까지 유지가 되는 애들. 오래 저장되는 애들.
>스택 영역, Stack Area
: 임시 저장의 영역
: 지역변수, 매개변수가 할당되는 영역
: 지역 변수, 매개변수들은 해당 지역을 벗어나면 사라지는 정보들이기 때문에. (메소드 종료 시 소멸)
>힙 영역, Heap Area
: 인스턴스가 저장되는 영역
: 인스턴스를 스택 영역이 아닌, 힙 영역에서 다루는 이유
: 인스턴스는 하나 이상의 참조 변수(스택 영역)가 참조할 수 있기 때문에 함부로 없앨 수 없다.
: 대신, 참조하는 변수가 아무것도 없을 때 자바 가상머신이 인스턴스를 지워준다.(가비지 컬렉션 대상)
: 이를 가비지 컬렉션이라고 하고, 조심스럽게 지우는 것이라는 의미로 파악해두자.
