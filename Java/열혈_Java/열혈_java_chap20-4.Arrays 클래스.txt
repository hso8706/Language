*자바의 기본 클래스

-Arrays 클래스
: Math 클래스처럼 인스턴스로 활용하는 것이 아니라 Arrays 클래스 내에 선언된 static 변수들과 static 메소드를 활용하기 위한 클래스
>copyOf, copyOfRange
: 배열을 복사하고 배열로 반환하는 메소드
>arraycopy
: 배열을 복사하여 빈 배열에 집어넣는 메소드, 미리 빈 배열 공간을 만들어놔야함.
>equals
: 배열의 내용을 비교하여 boolean으로 반환
: 인스턴스 배열 또한 오버라이딩 되어 있기 때문에 비교 가능
>equals 추가 내용; 인스턴스 배열의 비교
: 인스턴스 배열의 비교는 각 배열 인덱스를 비교한다.
: 인덱스 0, 인덱스 1 ... 인덱스 n까지 하나하나 Object.equals를 이용하여 비교함.
: Object.equals는 참조 값을 비교한다. 즉, 인스턴스 배열의 속내용이 같더라도 참조 값이 다르면 false를 반환한다
+equals 메소드를 오버라이딩해야 내용 비교가 된다.

-sort 메소드(Arrays 클래스)
>선언
public static void sort ( int[ ] a )
: 다른 기본형들 오버로딩 되어있음. (인스턴스 배열 포함)
>정의
: 매개변수 a로 전달된 배열을 오름차순으로 정렬
>compareTo 메소드( A . compareTo ( B ) )
: 인스턴스 배열의 sort 메소드를 실시할 때 적용하는 기준이 되는 메소드
: interface Compareable에 존재하는 추상 메소드 
: A(주체)가 B(인자)보다 크면 양수, 작으면 음수, 같으면 0반환
: compareTo 인터페이스를 구현하면서 어떤 것을 기준으로 할 것인지 제시해줘야함.
// 대소비교 if구문을 return A - B로 바꾸는 간결함!
// 반환 값이 양수, 음수, 0,  셋 중 하나면 되기 때문

-배열의 탐색(Arrays 클래스)
>선언
public static int binarySearch ( int[ ] a, int key )
: 다른 기본형들 오버로딩 되어있음. (인스턴스 배열 포함)
>정의
: 배열 a에서 key를 찾아서 있으면 key의 인덱스 값, 없으면 0보다 작은 수 반환
: 즉, 타겟을 배열에서 있는지 없는지 탐색하고 있으면 어디있는지까지 반환
>조건
: 이름처럼 이진 탐색을 기반으한 알고리즘으로 탐색을 실시함.
: 이진 탐색 알고리즘으로 탐색하기 위해서는 배열이 오름차순으로 정렬되어 있어야한다.
