-Java 실행 구조
>하드웨어
: 가장 기반이 되는 부분.
: 운영체제를 실행시킨다.
>운영체제(OS, Operating System)
: 프로그램을 구동시킨다.
: 종류가 다양함(Mac OS, Window, Linux)
>Java Virtual Machine
: 기본 프로그램들은 OS가 구동시킨다. 하지만 자바는 가상구동시스템을 거쳐서 구동된다.
: 운영체제에 의해 구동되는 소프트웨어
: OS가 달라도 Java를 구동시키기 위해 이같은 중간 시스템을 거친다.
: 하지만 각 OS마다 적합한 JVM을 설치해야 한다.
>Java
: 가상구동시스템에 의해 실행됨.

-Java 파일과 컴파일러
>소스파일
: .java 확장자를 가진 파일
>소스코드
: 소스파일에 저장된 프로그램 코드
>자바 컴파일러
: 소스파일에 저장된 소스코드를 자바 바이트코드로 전환시켜 자바 가상머신에서 실행될 수 있게 처리한다.
>자바 바이트코드
: .class 확장자를 가진 파일
: 자바 가상머신이 실제로 실행시키는 파일
: 자바 컴파일러에 의해 생성됨.

-println, print
>println : 마지막에 개행한다.
>print   : 마지막에 개행하지 않는다.

-CMD 작동법
1. cd file_path
// 를 통해 .java 파일이 존재하는 디렉토리로 이동
2. javac file_name.java
// 를 통해 컴파일 실시(읽을 수 있는 class 파일 변환 및 생성)
3. java file_name
// 확장자를 붙이지 않으면 알아서 실행.
